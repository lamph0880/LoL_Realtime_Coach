## 🏷️ 4. 어노테이션

### 어노테이션 도구
- **자체 제작 GUI 툴 사용**: PyQt6 기반의 맞춤형 데스크톱 애플리케이션
- 줌(Zoom), 마우스 드래그를 이용한 바운딩 박스 생성/크기 조절 및 단축키(A/D 이동, alt + z 되돌리기) 지원

<img width="720" height="480" alt="{91099217-C6C2-4B9D-9B02-B447116DE1C8}" src="https://github.com/user-attachments/assets/490f4f0e-0954-4a61-aecc-2be1e6aee036" />

### 포맷 방법
- **YOLO txt 포맷** 사용 (`클래스ID x_center y_center width height`)
- 좌표 및 크기는 이미지 전체 해상도에 비례하여 0부터 1 사이의 정규화된(Normalized) 값으로 변환되어 기록됨
- 이미지와 동일한 파일명을 가진 `.txt` 파일로 `data/labels/` 폴더에 각각 저장

### 검수 방법
- 제작된 에디터의 시각화 기능을 통해 저장된 라벨링 정보를 실제 이미지 위에 오버레이(Overlay)하여 실시간 점검
- 단축키와 마우스를 활용해 위치 어긋남이나 누락된 객체를 즉시 확인 및 수정(이동/크기 조정/삭제) 조치

## ⚠️ 7. 주의사항

- **객체 가림 현상 (Occlusion)**: 다른 UI 요소에 의해 챔피언 아이콘이나 미니맵이 일부 가려질 수 있습니다. 식별이 매우 어려운 경우 전처리로 제거하였으나 일부 포함될 수 있습니다.
- **클래스 불균형 (Class Imbalance)**: 통계에서 볼 수 있듯 아군(ORDER)과 적군(CHAOS) 간 데이터 수 비율 차이가 다소 존재합니다. 모델 학습 과정에서 이를 고려한 데이터 증강(Augmentation) 도 고려 요소입니다.
- **해상도 일관성**: 미니맵 관련 데이터와 챔피언 아이콘 관련 데이터의 원본 이미지 크기(560x560 vs 100x100)가 상이하므로, YOLO 네트워크 입력 크기(imgsz) 조절 시 해상도 손실에 유의해야 합니다.

## 1. LLM 호출 기준

### 1.1 자동 호출 조건

LLM(Gemini/Grok)은 다음 두 조건을 모두 만족할 때 자동으로 호출됩니다:

| 조건 | 상세 |
|------|------|
| **위험도 임계값** | 위험도 ≥ 65 (RISK_AUTO_ALERT) |
| **쿨타임** | 마지막 알림으로부터 ≥ 10초 경과 (RISK_ALERT_COOLDOWN) |

즉, 위험도 ≥ 65 이면서 마지막 알림으로부터 10초 이상 경과했을 때만 호출됩니다.

아무리 빈번하게 위험도 65 이상이 되어도 **10초마다 한 번씩만 호출**됩니다.

수동 호출 경로(F9)는 본 버전에서 폐기되었습니다. LLM 호출은 위 자동 트리거 경로로만 발생합니다.

## 2. 위험도 수치 범위 및 의미

### 2.1 범위

| 항목 | 값 | 색상 | 의미 |
|------|-----|------|------|
| **최솟값** | 0.0 | 🟢 Green | 안전 |
| **최댓값** | 100.0 | 🔴 Red | 위험 |

### 2.2 위험도 수치별 구간

| 범위 | 색상/상태 | LLM 호출 |
|------|----------|---------|
| 0~30 | 🟢 Green (안전) | ❌ 안 함 |
| 31~65 | 🟡 Yellow (주의) | ❌ 안 함 |
| 66~100 | 🔴 Red (위험) | ✅ 호출 |

### 2.3 위험도 계산 방식

**멀티팩터 기반 고도화** (integrated_yolo.py의 _calc_risk())

위험도는 다음 4가지 요소를 종합하여 계산합니다:

#### 2.3.1 기본: 거리 기반 포인트 (Distance-based Points)

미니맵에서 감지된 각 적 챔피언과 내 위치 사이의 거리를 계산:

| 거리 범위 | 상수명 | 포인트 |
|----------|------|--------|
| ≤ 60px | DANGER_RADIUS_HIGH | +40점/적 |
| 61~120px | DANGER_RADIUS_MID | +20점/적 |
| > 120px | 안전 범위 | +0점 |

**공식:** `distance_points = Σ(각 적의 거리 포인트)`

#### 2.3.2 수치적 우위/열위 (Numerical Advantage Factor)

팀 전투력 균형을 평가:

| 조건 | 우위도 배율 |
|------|----------|
| enemy_count > ally_count + 2 | 1.40 (큰 열위) |
| enemy_count = ally_count + 2 | 1.30 |
| enemy_count = ally_count + 1 | 1.15 (약간의 열위) |
| enemy_count = ally_count | 1.00 (동등) |
| ally_count > enemy_count | 0.85 (우위) |

**공식:** `advantage_factor = 우위도 배율`

예: ally=5, enemy=4 → advantage_factor=1.00  
예: ally=4, enemy=5 → advantage_factor=1.15

#### 2.3.3 위협 집중도 (Threat Concentration)

감지된 적들이 얼마나 밀집해 있는지 평가합니다.

**계산 방식:**
1. 적들의 중심점 좌표 평균을 계산
2. 중심으로부터 각 적까지의 거리 편차(standard deviation) 계산
3. 편차가 작을수록 집중된 위협 (높은 계수) → 편차가 클수록 분산된 위협 (낮은 계수)

| 집중도 | 조건 | 집중도 계수 |
|--------|------|----------|
| 매우높음 | 편차 < 30px | 1.25 |
| 높음 | 편차 30~60px | 1.15 |
| 중간 | 편차 60~100px | 1.00 |
| 낮음 | 편차 > 100px | 0.80 |

**주의:** 적이 2명 이하면 집중도 계수 = 1.00 (계산 불가)

#### 2.3.4 검출 신뢰도 가중치 (Detection Confidence Weight)

각 적 챔피언의 검출 신뢰도(top1_conf)를 포함:

| 신뢰도 범위 | 가중치 |
|-----------|--------|
| ≥ 0.90 | 1.00 (100% 기여) |
| 0.75~0.89 | 0.95 |
| 0.60~0.74 | 0.85 |
| < 0.60 | 0.70 |

**적용:** 각 적의 거리 포인트에 개별 가중치 적용 후 합산

**공식:** `weighted_distance_points = Σ(각 적의 거리 포인트 × 신뢰도 가중치)`

#### 2.3.5 최종 위험도 계산 (Final Risk Score)

```
최종_위험도 = min(100.0, weighted_distance_points × advantage_factor × concentration_factor)
```

**상한 제한:** 최댓값은 100.0으로 고정

**계산 예시:**

**예 1) 적 2명, 동등한 수치, 분산된 위협**
- ally=5, enemy=5
- 적1: 거리 50px (40점), 신뢰도 0.92 (1.00 가중치) → 40점
- 적2: 거리 80px (20점), 신뢰도 0.85 (0.95 가중치) → 19점
- weighted_distance_points = 40 + 19 = 59점
- advantage_factor = 1.00 (동등)
- concentration_factor = 1.00 (2명 이하)
- **최종_위험도 = min(100, 59 × 1.00 × 1.00) = 59.0**

**예 2) 적 3명, 수치 열위, 밀집된 위협**
- ally=4, enemy=6 → advantage_factor = 1.30
- 적3명 모두 55~65px 범위 (편차 ~15px) → concentration_factor = 1.25
- 적1: 거리 60px (40점), 신뢰도 0.95 (1.00 가중치) → 40점
- 적2: 거리 65px (40점), 신뢰도 0.88 (0.95 가중치) → 38점
- 적3: 거리 55px (40점), 신뢰도 0.90 (1.00 가중치) → 40점
- weighted_distance_points = 40 + 38 + 40 = 118점
- **최종_위험도 = min(100, 118 × 1.30 × 1.25) = min(100, 192.25) = 100.0**

## 4. 좌표 규칙

### 4.1 좌표 계층 구조

```
모니터 스크린 좌표
  ↓ (설정된 bbox로 캡처)
미니맵 이미지 (로컬 좌표)
  ↓ (YOLO 감지)
bbox 좌표 + 중심 좌표
```

### 4.2 각 계층의 원점과 규칙

| 계층 | 원점 (0,0) | 범위 | 사용 용도 |
|------|-----------|------|---------|
| **화면** | 좌상단 | 0~1920 | 미니맵 설정 |
| **미니맵 이미지** | 좌상단 | 0~W/H | YOLO bbox |
| **중심점** | 좌상단 | 0~W/H | 위험도 계산, LLM 입력 |

### 4.3 YOLO bbox 형식

**(x1, y1, x2, y2)** → 좌상단과 우하단 좌표

```
예: bbox = (30, 40, 50, 60)
  x1=30, y1=40 → 좌상단
  x2=50, y2=60 → 우하단
  중심 = ((30+50)/2, (40+60)/2) = (40, 50)
```

**모든 좌표는 미니맵 이미지 로컬 좌표입니다** (화면 좌표 아님)

## 5. 팀 분류 및 챔피언 감지 필터링

### 5.1 팀 분류 (Model F) - 빨간색/파란색 테두리

**알고리즘: HSV Ring 색 분석** (결정론적, ML 모델 없음)

1. cv2.HoughCircles로 ring 기하학적 검출
2. ring 둘레 ±3px 띠 영역만 색 분석 (챔피언 초상화 색 영향 제외)
3. 실패 시 fallback: 가장자리 영역 사용
4. HSV 색상으로 ally/enemy 다수결

#### 색상 범위 (HSV, 0~255 범위)

| 색상  | Hue 범위        | 팀         |
| --- | ------------- | --------- |
| 파란색 | 125~180       | ally (아군) |
| 빨간색 | 0~20, 230~255 | enemy (적) |

**검증 결과: 100/100 정확도** (data_0429 blue_border 50 + red_border 50)

### 5.2 챔피언 감지 필터링 (3단계 파이프라인)

#### Stage 1: Model A' (YOLO Detection) - 위치 감지

| 필터링 | 조건 |
|--------|------|
| conf_threshold | 신뢰도 ≥ 0.35만 보관 |
| iou_threshold | 박스 겹침 > 0.5인 경우 NMS로 제거 |

#### Stage 2: Model F (Team Classifier) - ally/enemy 결정

**필터링 없음** - 결정론적 색 분석으로 항상 ally 또는 enemy 할당

#### Stage 3: Model B (Champion Classifier) - 챔피언 분류

| 필터링 | 상태 |
|--------|------|
| 신뢰도 제한 | ❌ 없음 - 모든 예측 포함 (top1_conf ≥ 0) |
| 상태 | v2 검증 통과, 172개 클래스 분류 |

### 5.3 챔피언 감지 여부

✅ **있음** - 적 챔피언 이름과 신뢰도(conf)를 감지해서 LLM에 전송

Box 데이터 구조:
- `top1: str` - 감지된 챔피언명 (예: "Ahri")
- `top1_conf: float` - 신뢰도 (0~1)
